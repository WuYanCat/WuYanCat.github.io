Maven

#Maven
Maven是什么：
项目管理工具，对软件项目提供构建与依赖管理
Apache基金会的顶级开源项目
为Java项目提供了统一的管理方式，事实上已经成为业界标准。

Maven核心特点：
项目设置遵循统一的规则，保证不同开发环境的兼容性。
强大的依赖管理，项目依赖组件自动下载，自动更新。
可扩展的插件机制，使用简单，功能丰富。

配置一下环境变量。。。
在idea的设置里面更改一下maven版本

Maven依赖管理：
依赖坐标系：
GroupId：机构或团体的英文，采用“逆向域名”形式书写
AritifactId：项目名称，说明其用途。例如：cms，oa
Version：版本号，一般为“版本+单词”的形式。
重要！！！maven的搜索界面：search.maven.org
直接上maven的搜索界面进行搜索下载。先复制其坐标系，在回到pom.xml文件创建,所有的引用都放在这里面，然后右上角更新，下载库。
阿里云镜像仓库，解决国外网速问题，网址：maven.aliyun.com在xml文件里使用标签引用，这样一来，在下载文件时，优先从相应的镜像仓库里面查找。找不到再从远端查找

maven生命周期：执行某个命令时会将前面的命令一起执行了
mvn + …
validate 验证项目是否正确且所有必须信息是可用的
compile 源代码编译阶段
test 运行test目录下的测试代码验证src下源代码的逻辑
package 生成产出物jar，war文件
verify 运行任意的检查来验证项目包有效且达到质量标准
install 安装打包的项目到本地仓库，以供其它项目使用
deploy 拷贝最终的工程包到远程仓库中，以共享给其它开发人员和工程。

maven插件技术：
提供功能扩展的能力

所谓FatJar是指将所有依赖包装为独立可运行的Jar

html

HTML：
1.标记又被称为标签
2.xml的标签是自定义的
3.html的标签是内置的
HTML骨架标签：
html：根标签，除DOCTYPE外的其它内容都要卸载html标签内。
head：页面头部，页面中非常重要但是又不会显示出来的代码写在此处，如css，js
title：页面标题，显示在浏览器的标题栏
body：页面主题，需要显示在浏览器的内容都写在此处
注：html是一种松散的组织结构
标签结构不正确或没有写在规定位置也能显示出来
在编写时要按照要求来
HTML常用标签：
1.排版标签：
标题标签：h1~h6，h1最大，独占一行
属性包括：align设置内容的位置，包括left，central，right
水平线和换行：hr标签是水平线，br换行
段落标签：p标签，也有align属性，独占一行
html实体：&nbsp；相当于一个空格&emsp；全角空格，&lt；相当于<
div标签：网页布局，与css配合使用，目前一行只能放一个
span标签：将文本独立，可独立设置样式
字符格式化标签：b/strong标签加粗，i/em斜体，u/ins标签下划线，del/s标签删除线
图片标签：img标签引入图片，src属性设置路径，alt属性设置在出错情况下的显示字符串，width/height标签设置宽高
链接标签：a标签包括超链接和锚链接，使用属性href指定跳转的页面/锚“#。。。”，在其它标签内使用id表示锚点，使用target指定链接的打开方式，_blank表示在新窗口 打开，_self表示当前页面（默认）
2.表格
table制作表格边框
嵌套tr表示行，td表示列
属性：border设置边框像素，width/height表示表格宽高，cellspacing设置单元格之间，单元格与边框的间隙，cellpadding设置单元格内容与单元格的间隙，align设置表格在页面中的位置，bgcolor设置背景颜色，可以用单词或十六进制表示
表格合并：使用colspan属性实现列合并，rowspan实现行合并
其余标签：caption表格标题，thead头部标签，tbody表格主体，tfoot表格脚部
3.列表：
有序列表：ol和li标签
无序列表：ul和li标签
自定义列表：dl，dt，dd三种标签
4.表单：
form表单外层，method属性指定表单提交方式，action属性指定表单提交位置
文本框：input type=”text”,密码框 input type=”password”
常用属性：name起名，value为表单域的值，placeholder设置框中默认值，maxlength设置最大长度，readonly只读，disabled禁用
单选框input标签内type属性设置为radio，同时设置相同的name，复选框type设置为checkbox，name名称需要加上[]
文件域type属性设置为file，multiple属性设置允许同时选中多个文件，type属性设置为button为普通按钮，reset为重置表单，submit为表单提交。
tyoe设置为hidden为隐藏域。
下拉菜单：select与option标签嵌套
文本域：使用textarea标签，cols和rows设置列数与行数

PYQT5

1.毕设简介
制作一个可执行的数据库文件，可以保存大量的焊接数据，点击放置在桌面的快捷方式能够进入一个弹出的窗体，在该窗体及之后的几个窗体上实现对焊接数据的增删改查操作。
实现要点：
1.登陆界面：该界面实现对用户的识别操作，预计设计两个不同的操作权限，较高权限的用户可以对数据进行增删改查操作，而权限较低的用户只可以对数据进行查询操作。由于将整个文件打包封装，因此设计者可以直接打开源代码进行相应的修改功能，而无需进入登陆界面。
我通过在数据库内增加一个用户表，登陆时需要查找该用户表，在用户表内存在相应的用户数据就可以登陆。
2.主页面：该页面实现对数据操作的封装，相当于一个目录，在该页面上可以选择需要进行的操作，之后在弹出的界面中进行相应的操作
3.各操作页面，实现对数据的各种相关操作。
2.语言选择
1.mfc 使用c++编写的框架，需要对c++有一定了解，技术比较过时。
2.fx和swing 为java框架，本人对Java语言较为熟悉，但是Java对于可视化界面的操作不太流行，相关资料较少，技术发展缓慢。
3.vb 为微软提供的操作语言，可以直接实现可视化编程，相当于一个直接操作的界面，需要重新学习vb语言规则，而且我不想学。
4.pyqt5和pyside2 qt语言是以c++编写的可视化图形界面编程的一个模块，在各种语言里都被引用，在python中被称为pyqt，导入该模块，即可使用相应的函数实现相应图形的绘制。pyqt5和pyside2相当于一个是民间组织而一个是官方支持，内容几乎相同，学习时可以不分彼此，而且在pyqt里内置了qt designer插件，也可以实现以图形的方式实现编程，可以将绘制好的图形转换为相应的python代码，而程序猿只需要对其进行相应的修改，增加需要实现的功能，而图形绘制的繁琐操作就可以忽略。
3.目前进展
已经设计好登陆页面的全部功能，初步设计出一个主页面，但只实现了查询用户的功能，用户维护页面仅实现了基本的显示功能，尚未与数据库进行连接。

@Test
public void testSqlSessionFactory() throws IOException {
    //利用Reader加载classpath下的mybatis-config.xml核心配置文件
    Reader reader = Resources.getResourceAsReader("mybatis-config.xml");
    //初始化SqlSessionFactory对象,同时解析mybatis-config.xml文件
    SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);
    System.out.println("SessionFactory加载成功");
    SqlSession sqlSession = null;
    try {
        //创建SqlSession对象,SqlSession是JDBC的扩展类,用于与数据库交互
        sqlSession = sqlSessionFactory.openSession();
        //创建数据库连接(测试用)
        Connection connection = sqlSession.getConnection();
        System.out.println(connection);
    }catch (Exception e){
        e.printStackTrace();
    }finally {
        if(sqlSession != null){
            //如果type="POOLED",代表使用连接池,close则是将连接回收到连接池中
            //如果type="UNPOOLED",代表直连,close则会调用Connection.close()方法关闭连接
            sqlSession.close();
        }
    }
}

  • MyBatisUtils工具类，创建全局唯一的SqlSessionFactory对象

public class MyBatisUtils {
//使用静态static属于类而不属于对象，实现全局唯一
private static SqlSessionFactory sqlSessionFactory=null;
static {//static块可用于初始化静态对象
Reader reader = null;
try {
reader = Resources.getResourceAsReader(“mybatis-config.xml”);
sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);
} catch (IOException e) {
e.printStackTrace();
//初始化错误时，通过抛出异常通知调用者
throw new ExceptionInInitializerError(e);
}
}

/**
 * 创建一个新的SqlSession对象
 * @return
 */
public static SqlSession openSession(){
    return sqlSessionFactory.openSession();
}

/**
 * 关闭SqlSession对象
 * @param sqlSession
 */
public static void closeSession(SqlSession sqlSession){
    if(sqlSession!=null){
        sqlSession.close();
    }

}

}

9.selectKey与useGeneratedKeys的区别
都是在insert标签内使用
前者为显示获取，后者则根据当前驱动自动获取，隐式获取
前者适用于所有关系型数据库，后者只适用于支持“自增主键”类型的数据库，如oracle就不行。
前者适用广，但编写麻烦；后者简单但应用有限。

JDK8开始支持，用来让程序编写更优雅
可以更简洁的实现匿名内部类与函数声明与调用
基于Lambda提供的stream流式处理极大的简化对集合的操作
2.语法格式
(参数列表)->实现语句
参数列表可以使用逗号分隔，参数类型可以省略，单参数括号可以省略
实现语句如果是单行可以直接写，多行则用{}包裹

public interface MathOperation {
public Float operate(Integer a,Integer b);
}
public class LambdaSample {
public static void main(String[] args) {
    //1.标准lambda使用方式
    //约束条件，接口只有一个抽象方法，也被成为函数式接口
    MathOperation addition = (Integer a,Integer b)->{
        System.out.println("加法");
        return a+b+0f;
    };
    System.out.println(addition.operate(5,3));
    //2.Lambda允许忽略参数类型
    MathOperation substraction = (a,b)->{
        return a-b+0f;
        };
    System.out.println(substraction.operate(5,3));
    //3.单行实现代码可以省略大括号和return
    MathOperation multiplication = (a,b)->a*b+0f;
    System.out.println(multiplication.operate(5,3));
    //4.若只有一个参数，则小括号都可以省略
    }
}

原文题目：Virtual Reality Robot-Assisted Welding Based
on Human Intention Recognition
翻译题目：基于人类意图识别的虚拟现实机器人辅助焊接

安装了定制钨极气体保护焊(GTAW)焊炬的6自由度工业机器人UR-5作为焊接工作的最终执行者;
重点是识别机器人应该执行的命令人的意图，将人手的移动情况作为主要收集的数据；
使用分层架构的信息物理系统CPS来增强焊接操作，便于机器人由其操作人员的指挥
使用虚拟现实技术创建远程操作环境以防止环境对人产生的危害
使用Baum-Welch算法训练隐马尔可夫模型HMM，贝叶斯信息准则BIC用于确定隐藏状态的数量；
使用正向算法估计状态占据概率分布；
本文目标：
人机协作，让机器人能够更好的识别人的意图，按照人的思维进行操作，同时对人的无意义的错误操作进行纠正，使机器人具有一定程度上的自主性；
在焊接过程中，机器人的作用主要是进行危害性大，重复性的工作，保护人类工作者免受伤害，同时纠正少部分由于人类工作者技术生疏或意外导致的明显的错误；
人类工作者的作用主要是针对机器人难以理解，但操作者易于理解的部分，对机器人进行实时的调整，以实时适应复杂的焊接过程。
将需要密集劳动的部分交给机器人，实时适应的部分交给人类工作者，应当可以极大改善工作环境，提高工作效率。

词语解释：
蒙特卡罗方法：使用随机数解决问题，相对的是确定性算法
隐马尔可夫模型(HMM)：马尔可夫链的一种，它的状态不能直接观察到，但能通过观测向量序列观察到，每个观测向量都是通过某些概率密度分布表现为各种状态，每一个观测向量是由一个具有相应概率密度分布的状态序列产生。所以，隐马尔可夫模型是一个双重随机过程—-具有一定状态数的隐马尔可夫链和显示随机函数集
隐马尔可夫模型（HMM）可以用五个元素来描述，包括2个状态集合和3个概率矩阵

  1. 隐含状态 S
    这些状态之间满足马尔可夫性质，是马尔可夫模型中实际所隐含的状态。这些状态通常无法通过直接观测而得到。（例如S1、S2、S3等等)
  2. 可观测状态 O
    在模型中与隐含状态相关联，可通过直接观测而得到。(例如O1、O2、O3等等，可观测状态的数目不一定要和隐含状态的数目一致。）
  3. 初始状态概率矩阵 π
    表示隐含状态在初始时刻t=1的概率矩阵，(例如t=1时，P(S1)=p1、P(S2)=P2、P(S3)=p3，则初始状态概率矩阵 π=[ p1 p2 p3 ].
  4. 隐含状态转移概率矩阵 A。
    描述了HMM模型中各个状态之间的转移概率。
    其中Aij = P( Sj | Si ),1≤i,,j≤N.
    表示在 t 时刻、状态为 Si 的条件下，在 t+1 时刻状态是 Sj 的概率。
  5. 观测状态转移概率矩阵 B （英文名为Confusion Matrix，直译为混淆矩阵不太易于从字面理解）。
    令N代表隐含状态数目，M代表可观测状态数目，则：
    Bij = P( Oi | Sj ), 1≤i≤M,1≤j≤N.
    表示在 t 时刻、隐含状态是 Sj 条件下，观察状态为 Oi 的概率。
    总结：一般的，可以用λ=(A,B,π)三元组来简洁的表示一个隐马尔可夫模型。隐马尔可夫模型实际上是标准马尔可夫模型的扩展，添加了可观测状态集合和这些状态与隐含状态之间的概率关系。
    前向算法（应用于评估问题）：针对某一观测序列，使用已知的HMM模型进行推到，计算各模型产生该结果的概率，从中选出最优解。如语音识别。
    Viterbi算法（应用于解码问题）：给定观测序列 O=O1O2O3…Ot 和模型参数λ=(A,B,π)，怎样寻找某种意义上最优的隐状态序列采用，推断其隐状态序列，如中文分词，发展中国家分词为“发展-中-国家”还是“发展-中国-家”
    前向后向算法（学习问题）：即HMM的模型参数λ=(A,B,π)未知，如何调整这些参数以使观测序列O=O1O2O3…Ot的概率尽可能的大。通常使用Baum-Welch算法以及Reversed Viterbi算法解决。
    马尔科夫链：是概率论和数理统计中具有马尔可夫性质（Markov property）且存在于离散的指数集（index set）和状态空间（state space）内的随机过程（stochastic process）
    马尔可夫性质：未来仅依靠于现在的状态而与过去无关，在给定现在状态时，它与过去状态（即该过程的历史路径）是条件独立的，那么此随机过程即具有马尔可夫性质。具有马尔可夫性质的过程通常称之为马尔可夫过程。
    贝叶斯决策理论：是主观贝叶斯派归纳理论的重要组成部分。是在不完全情报下，对部分未知的状态用主观概率估计，然后用贝叶斯公式对发生概率进行修正，最后再利用期望值和修正概率做出最优决策。

本文内容：
第一部分：简介
我们使用基于HMM的状态占据概率的概念来模拟不稳定的随机性，并应用基于HMM的方法来识别人的意图，使用人手移动速度作为可观察变量。对于我们提出的方法，焊接任务不能预先分解为子任务。因此，隐藏状态号不能被预先识别。为了解决这个问题，我们提出使用贝叶斯信息准则(BIC)来识别隐状态的数量。使用Baum-Welch算法训练HMM，然后使用前向算法估计隐藏状态占据概率分布。机器人执行由可观察变量的统计期望确定的预期动作。实验表明，在机器人的帮助下，通过识别人类的意图，人类可以更好地完成焊接任务。这验证了虚拟现实机器人辅助焊接系统的有效性。
第二部分：系统配置
提出的机器人辅助焊接系统的信息物理模型具备四个空间，包括用户空间U、虚拟空间V、机器人空间R和工具空间T与分层架构共存。通过虚拟空间V和机器人空间R，用户空间U和工具空间T中的对象相互映射。
系统具备的特性：空间分离性，身临其境的能力，操作能力。
第三部分：HMMS原则
HMM 是序列数据的统计模型，其中潜在状态序列是马尔可夫链，其值决定了相应可观察变量的分布。在我们的 HMM 中，人手的移动速度 ν = [νx , νy , νz] T 是可观察变量。
第四部分：模型识别
复杂的计算。。。EM算法估计HMM参数，BIC估计隐藏状态数量。
第五部分：人类意图识别
对人手在三维方向上的移动进行分析，在沿着x轴的主要行进方向上，所识别的预期行进速度实现了接近真实速度的平均值和减小的标准偏差。在横向y轴和垂直z轴方向，也实现了接近零的平均值和更低的标准偏差。z方向发生了一个有趣的现象。原始速度数据具有最大的方差，但是预期速度数据具有最小的方差。这表明z方向的运动具有最高的随机性，即与其他两个方向相比，z方向的运动没有明显的人类意图转变。
第六部分：机器人辅助焊接实验
在使用或者不使用基于HMM的IR算法下进行实验，对比发现在使用相应算法的情况下，人类操作者的表现更好。
